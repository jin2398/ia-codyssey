# Mission Computer Log Analysis

## 1. 분석 대상
- 로그 파일: mission_computer_main.log

## 2. 로그 분석 과정
- Python을 사용하여 로그 파일을 읽고 분석하였다.
- 로그를 시간 역순으로 확인하여 최근 이벤트를 먼저 분석하였다.
- 특정 키워드를 기반으로 문제 로그를 탐지하였다.

## 3. 문제 발생 로그
- 2023-08-27 11:35:00,INFO,Oxygen tank unstable.
- 2023-08-27 11:40:00,INFO,Oxygen tank explosion.

## 4. 사고 원인 분석
로그 분석 결과 11:35에 "Oxygen tank unstable"(산소 탱크 불안정) 메시지가 기록되었으며 
 이후 11:40에 "Oxygen tank explosion"(산소 탱크 폭발) 메시지가 발생하였다. 
 따라서 산소 탱크의 불안정 상태가 폭발로 이어진 것으로 판단된다.

## 5. 결론
산소 탱크의 불안정 상태가 폭발 사고의 주요 원인으로 분석된다.
 향후 유사 사고를 방지하기 위해 산소 탱크 상태 모니터링과 시스템 안전 점검이 필요하다.
